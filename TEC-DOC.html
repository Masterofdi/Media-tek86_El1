Media_tek86

Présentation générale

Media_tek86 est une application Windows Forms conçue pour gérer deux entités principales : le personnel et leurs absences. Elle permet également d’afficher le contenu de quelques tables annexes (responsable, motif, service) de manière générique.

Fonctionnalités principales

  • Affichage de données
    Sélection d’une table (personnel, absence, responsable, motif, service) pour afficher toutes ses lignes dans une grille.
  • Ajout de personnel
    Création d’un nouvel enregistrement dans la table personnel en saisissant nom, prénom, service, e-mail et téléphone.
  • Modification
    Mise à jour des informations existantes pour un employé ou une absence directement depuis la grille ou via de petites boîtes de dialogue de saisie.
  • Suppression
    Suppression d’un salarié ou d’une absence après confirmation de l’utilisateur.
  • Affichage générique
    Consultation rapide du contenu des tables responsable, motif et service, sans fonctionnalités CRUD spécifiques.

Organisation du projet

L’application s’appuie sur une architecture en couches :

  • IHM Windows Forms
    Une seule fenêtre principale avec :
    • Un menu déroulant pour choisir la table à afficher.
    • Une grille (DataGridView) qui liste les enregistrements.
    • Des boutons « Ajouter », « Modifier » et « Supprimer » selon la table sélectionnée.
  • Couche Modèle
    Des classes simples représentant un employé (Personnel) ou une absence (Absence), sans logique métier complexe.
  • Couche Accès aux données (DAL)
    Des classes spécialisées qui communiquent avec la base MySQL pour :
    • Récupérer toutes les lignes d’une table.
    • Créer, mettre à jour ou supprimer un enregistrement dans les tables personnel et absence.
    • Lire le contenu de tables annexes (responsable, motif, service) de façon générique.

Flux d’utilisation

  1. Au démarrage, l’application affiche la liste des employés dans la grille.
  2. L’utilisateur sélectionne une table dans le menu déroulant pour voir son contenu.
  3. Si la table est « personnel », il peut cliquer sur « Ajouter » pour créer un nouvel employé, « Modifier » pour mettre à jour ses informations ou « Supprimer » pour le retirer.
  4. Si la table est « absence », il peut cliquer sur « Modifier » ou « Supprimer » pour gérer ces enregistrements.
  5. Pour les tables « responsable », « motif » ou « service », la grille se contente d’afficher toutes les lignes, sans bouton d’édition.

Évolutions possibles

  • Ajouter un formulaire dédié pour saisir/modifier plusieurs champs en une seule fois plutôt que de petites boîtes de dialogue.
  • Ajouter une fonctionnalité « Créer une absence » en proposant de choisir un employé, une plage de dates et un motif.
  • Mettre en place un filtrage avancé (par service, par période d’absence).
  • Internationnaliser les textes via des ressources pour supporter plusieurs langues.
